# Algorithm Summary for `standalone.py`

This document summarizes the complete computational pipeline implemented in
`standalone.py`, following [BMSZ] (arXiv:1712.05552v6) and [BMSZb] (arXiv:2205.05266v4).

## Input

- **Dual partition** Ǒ: a Young diagram with good parity, given as a tuple of
  row lengths in decreasing order, e.g., `(4, 2)`.
- **Root system type** ★ ∈ {B, C, D, M(=C̃)}.

## Step 1: Orbit → Bipartition

**Function**: `orbit_to_bipartition(Ǒ, ★)`
**Reference**: [BMSZb] Section 2.8, equation (2.16).

Attach a pair of Young diagrams (ι_Ǒ, j_Ǒ) to Ǒ, given by column lengths.
The formulas depend on ★:

### ★ = B
Ǒ is type C (even rows). Let r₁ ≥ r₂ ≥ ... be the row lengths.
- c₁(j_Ǒ) = r₁/2
- For i ≥ 1: (c_i(ι_Ǒ), c_{i+1}(j_Ǒ)) = (r_{2i}/2, r_{2i+1}/2)

### ★ = M (= C̃)
Ǒ is type C (even rows).
- For i ≥ 1: (c_i(ι_Ǒ), c_i(j_Ǒ)) = (r_{2i-1}/2, r_{2i}/2)

### ★ = C
Ǒ is type B (odd rows).
- For i ≥ 1: (c_i(j_Ǒ), c_i(ι_Ǒ)) depends on whether the ★-pair (2i-1, 2i)
  is vacant, tailed, or balanced in Ǒ.

### ★ = D
Ǒ is type D (odd rows).
- c₁(ι_Ǒ) = (r₁+1)/2 if r₁ > 0
- For i ≥ 1: (c_i(j_Ǒ), c_{i+1}(ι_Ǒ)) depends on whether (2i, 2i+1)
  is vacant, tailed, or balanced.

## Good Parity

A dual partition Ǒ has **good parity** with respect to ★ if all its nonzero
row lengths have the correct parity:

| ★ | Ǒ orbit type | Row parity | Total parity |
|---|-------------|-----------|-------------|
| B | type C | all rows even | even |
| M (= C̃) | type C | all rows even | even |
| C | type B | all rows odd | odd |
| D | type D | all rows odd | even |

Reference: [BMSZb] Section 2.8, good parity assumption.

## Step 2: Primitive Pairs

**Function**: `primitive_pairs(Ǒ, ★)`
**Reference**: [BMSZb] Definition 2.21.

A ★-pair (i, i+1) is a pair of consecutive positive integers such that:
- i is odd if ★ ∈ {C, M}
- i is even if ★ ∈ {B, D}

A ★-pair (i, i+1) is **primitive** in Ǒ if r_i(Ǒ) - r_{i+1}(Ǒ) is positive and even.

PP_★(Ǒ) = set of all primitive ★-pairs in Ǒ.

## Step 3: W-representations (Coherent Family)

**Function**: `dpart2Wrepns(Ǒ, ★)`
**Reference**: [BMSZb] Section 8.3.

Enumerate all W-representations (bipartitions) in the coherent family attached
to Ǒ. For the special shape (℘ = ∅), the bipartition is (ι_Ǒ, j_Ǒ) directly.
For each subset of primitive pair indices, a different bipartition is generated
by "swapping" the column assignments.

The number of bipartitions = 2^{|PP_★(Ǒ)|}.

## Step 4: DRC Diagrams (Painted Bipartitions)

**Function**: `dpart2drc(Ǒ, ★)` → `fill_drc(τ, ★)`
**Reference**: [BMSZ] Definition 3.1 / [BMSZb] Definition 2.24.

A painted bipartition τ = (ι, P) × (j, Q) × γ where:
- (ι, P) and (j, Q) are painted Young diagrams
- γ ∈ {B⁺, B⁻, C, D, C̃}
- P and Q have identical sets of •-boxes

### Symbol constraints by type γ

| γ    | P symbols      | Q symbols      |
|------|---------------|---------------|
| B⁺/B⁻ | {•, c}      | {•, s, r, d}  |
| C    | {•, r, c, d}  | {•, s}        |
| D    | {•, s, r, c, d} | {•}         |
| M=C̃ | {•, s, c}     | {•, r, d}     |

### Painting constraints (Definition 3.1)

1. Removing {d}, {c,d}, {r,c,d}, or {s,r,c,d} boxes leaves a valid Young diagram
2. Each row: at most one s and at most one r
3. Each column: at most one c and at most one d

### Filling pipeline

DRC diagrams are generated by a layered filling algorithm.
Each step fills one type of symbol, proceeding from innermost to outermost:

| ★ | Step 1 (outermost) | Step 2 | Step 3 | Step 4 (innermost) |
|---|-------|--------|--------|--------|
| D | fill_c 'd' on L | fill_c 'c' on L | fill_r 'r' on L | fill_rdot 's' |
| C | fill_c 'd' on L | fill_c 'c' on L | fill_r 's' on R | fill_rdot 'r' on R |
| B | fill_c 'd' on R | fill_c 'c' on R | fill_r 'r' on R | fill_rdot 's' on L |
| M | fill_c 'd' on R | fill_c 'c' on R | fill_r 'r' on R | fill_rdot 's' on R |

Here "on L" means the operation acts on the left diagram (P), and "on R" means
the right diagram (Q). The SWdgm transform swaps L↔R before/after the fill.

## Step 5: Signature and Epsilon

**Functions**: `signature(τ, γ)`, `epsilon(τ, γ)`
**Reference**: [BMSZ] equations (3.3), (3.5), (3.6).

### Signature (p_τ, q_τ)

Each cell in the DRC diagram contributes to (p, q) as follows:

| Symbol | (p, q) contribution |
|--------|:------------------:|
| • (dot) | (1, 1) |
| c | (1, 1) |
| d | (1, 1) |
| r | (2, 0) |
| s | (0, 2) |

For type B, an additional overall contribution:
- B⁺: +(1, 0)
- B⁻: +(0, 1)

Equivalently:
```
p_τ = (#•) + 2(#r) + (#c) + (#d) + (#B⁺)
q_τ = (#•) + 2(#s) + (#c) + (#d) + (#B⁻)
```

For ★ ∈ {C, M}: all symbols contribute (1,1), so p_τ = q_τ = |τ| (total number of boxes).

### Group G_τ

```
G_τ = O(p_τ, q_τ)    if ★ = B or D
    = Sp_{2|τ|}(ℝ)    if ★ = C
    = M̃p_{2|τ|}(ℝ)   if ★ = M
```

### Dimension

```
dim τ = 2|τ| + 1   if ★ = B
      = 2|τ|       otherwise
```

### Epsilon ε_τ ∈ Z/2Z

```
ε_τ = 0  ⟺  symbol 'd' occurs in the first column of P or Q
```

## Step 5b: Tail of a Painted Bipartition (for ★ ∈ {B, D})

**Reference**: [BMSZb] Section 10.5, equation (10.7).

For ★ ∈ {B, D}, define k := (r₁(Ǒ) - r₂(Ǒ))/2 + 1, which is a positive integer.

### Tail orbit Ǒ_t

- If ★ ∈ {B, D}: Ǒ_t consists of two rows with lengths 2k-1 and 1.
- The tail type ★_t = D (so Ǒ_t has good parity w.r.t. D).
- PP_{★_t}(Ǒ_t) = ∅.

### Tail τ_t

Every element in PBP_{★_t}(Ǒ_t) has the form of a single-column painted
bipartition with entries x₁, x₂, ..., x_k from top to bottom:

```
  |x₁|
  |x₂|
  |..|  × ∅ × D    (for ★ = B or D)
  |xk|
```

### Tail multiset {x₁, ..., x_k}

**★ = B** (c₁(ι) ≤ c₁(j)):

The multiset {x₁, ..., x_k} = { Q(j,1) | c₁(ι)+1 ≤ j ≤ c₁(j) } ∪ {extra}

where extra =
- {c}  if α = B⁺ and (c₁(ι)=0 or Q(c₁(ι),1) ∈ {•,s})
- {s}  if α = B⁻ and (c₁(ι)=0 or Q(c₁(ι),1) ∈ {•,s})
- {Q(c₁(ι),1)} otherwise

**★ = D** (c₁(ι) > c₁(j) when |Ǒ| > 0):

The multiset = { P(j,1) | c₁(j)+2 ≤ j ≤ c₁(ι) } ∪ {extra}

where extra =
- {d}             if |Ǒ| = 0
- {c}             if r₂ = r₃ > 0, P(c₁(ι),1) ∈ {r,d}, and
                   (P(c₁(j)+1,1), P(c₁(j)+1,2)) = (r,c)
- {P(c₁(j)+1,1)}  otherwise

### Tail symbol x_τ

For ★ ∈ {B, D}:  x_τ := P_{τ_t}(k, 1), the symbol in the LAST box of the tail.

x_τ ∈ {c, d, r, s} determines the "type" of the tail, used in:
- Recursive counting (Proposition 10.11): f = f^{d} + f^{c,r} + f^{s}
- Descent image (Proposition 10.8): when r₁ = r₂, descent image = {τ' | x_{τ'} ≠ s}

## Step 6: Descent

**Function**: `descent(τ, ★)`
**Reference**: [BMSZ] Definition 3.14, Lemma 3.7, 3.10, 3.12.

The descent map ∇: PBP_★(Ǒ) → PBP_{★'}(Ǒ') where ★' is the Howe dual:

| ★ | ★' (Howe dual) |
|---|----------------|
| B | M (= C̃) |
| C | D |
| D | C |
| M | B |

### Naive descent (Lemma 3.7)

**For ★ ∈ {B, C}** (Lemma 10.4 of [BMSZb]):
- Keep ι, remove first column of j
- P'(i,j) = • or s if P(i,j) ∈ {•, s}; else P(i,j)
- Q'(i,j) = • or s if Q(i, j+1) ∈ {•, s}; else Q(i, j+1)

**For ★ ∈ {M, D}** (Lemma 10.5 of [BMSZb]):
- Remove first column of ι, keep j
- P'(i,j) = • or s if P(i, j+1) ∈ {•, s}; else P(i, j+1)
- Q'(i,j) = • or s if Q(i,j) ∈ {•, s}; else Q(i,j)

The "• or s" ambiguity is resolved by the interlacing condition.

### γ' determination (equation 3.11 / 10.5)

```
γ' = B⁺  if γ = C̃ and c does NOT occur in first column of (ι, P)
   = B⁻  if γ = C̃ and c DOES occur in first column of (ι, P)
   = ★'  otherwise
```

### Special corrections (Definition 3.14)

**Lemma 3.10** (★ = B⁺ special):
- Conditions: γ = B⁺, r₂(Ǒ) > 0, Q(c₁(ι), 1) ∈ {r, d}
- Action: P'(c₁(ι'), 1) := s

**Lemma 3.12** (★ = D special):
- Conditions: γ = D, r₂(Ǒ) = r₃(Ǒ) > 0, (P(c₂(ι),1), P(c₂(ι),2)) = (r,c),
  P(c₁(ι),1) ∈ {r, d}
- Action: P'(c₁(ι'), 1) := r

### Dual descent of ℘ (equation 3.15)

Shift all pair indices down by 1, and discard (0, 1):
```
℘' = ∇̃(℘) = {(i-1, i) : (i, i+1) ∈ ℘, i ≥ 2}
```
Equivalently: `℘' = {(i, i+1) : (i+1, i+2) ∈ ℘, i ≥ 1}`.

## Step 7: Descent Chain

**Function**: `descent_chain(τ, ★, Ǒ)`

Iteratively apply descent: τ → ∇(τ) → ∇²(τ) → ... until |Ǒ| = 0.
Each step records (drc, ★, signature, ε, Ǒ).
The types alternate: C ↔ D, M ↔ B.

## Step 8: Associated Cycle / Local System (MYD)

**Function**: `compute_AC(τ, ★, Ǒ)`
**Reference**: [BMSZ] equation (11.2), Section 9.

### Marked Young Diagram (MYD)

A marked Young diagram E: N⁺ → Z × Z, i ↦ (p_i, q_i), with finite support.
Parity constraints (Definition 9.3):
- ★ ∈ {B, D}: p_i = q_i when i is even
- ★ ∈ {C, C̃}: p_i = q_i when i is odd

### MYD Signature

Each MYD entry `(±p_i, ±q_i)` at level i contributes `(|p_i|, |q_i|)` to the
signature, analogous to the DRC per-cell formula. The sign (±) encodes
trivial vs non-trivial local system; the absolute value determines the
signature contribution.

Formally, via map V (equation 9.13): V(a,b) = (|a|, |b|), then formula (9.10):
```
Sign(E) = Σ_{k≥1} (k·|p_{2k}| + k·|q_{2k}|,  k·|p_{2k}| + k·|q_{2k}|)
        + Σ_{k≥1} (k·|p_{2k-1}| + (k-1)·|q_{2k-1}|,  (k-1)·|p_{2k-1}| + k·|q_{2k-1}|)
```

**Key property**: Sign(L_τ) = (p_τ, q_τ). Verified for all D, C, M types.

### Four operations on MYD (Section 9.4)

1. **Sign twist** ⊗(ε⁺, ε⁻) (9.15): for ★ ∈ {B, D}, flips signs at odd levels
2. **Involution T** (9.16): for ★ ∈ {C, C̃}, negates at i ≡ 2 (mod 4)
3. **Augmentation** ⊕(p₀, q₀) (9.18): insert new entry at level 1, shift others
4. **Truncation** Λ_{(p₀,q₀)} (9.20): subtract (p₀,q₀) from level 1 entry.
   Requires condition (9.19): E ⊒ (p₀, q₀).

### Theta lift of MYD (Section 9.5)

**For ★ ∈ {B, D}** — formula (9.29):
```
ϑ̂(E') = (T^{t_exp}(Λ_{(δ/2, δ/2)}(E'))) ⊕ (p₀, q₀)
```
where:
- δ = |s'| - |∇_naive(O)| (from Lemma 4.4, using SYD shift (9.11))
- (p₀, q₀) = (p,q) - (p',q') - ˡSign(O') + (δ/2, δ/2)
- t_exp = (p-q+1)/2 for B, (p-q)/2 for D
- ˡSign(O') computed from descended SYD via Lemma 9.2

**For ★ ∈ {C, C̃}** — formula (9.30):
```
ϑ̂(E') = T^{t_exp}(Σ_{j=0}^{δ} Λ_{(j, δ-j)}(E') ⊕ (n₀, n₀))
```
where:
- n₀ = (c₁(O) - c₂(O))/2
- t_exp = (p'-q')/2 for C, (|p'-q'|-1)/2 for C̃
- Sum only over j where truncation condition (9.19) is satisfied

### Signed Young Diagram (SYD) computation

**Function**: `_orbit_signed_yd(O_rows, p, q, ★)`

For orbit O with row lengths and signature (p,q):
- Even levels (B/D) or odd levels (C/M): forced p_i = q_i
- Free levels: assign (p_i, q_i) with p_i+q_i = ι(i) to match Σ(p_i-q_i) = p-q
- Process free levels smallest-to-largest

### ˡSign computation

**Function**: `_compute_lsign_from_orbit(...)`

1. Build SYD O from orbit row lengths and target (p,q)
2. Compute descended SYD O' via Lemma 9.2:
   - ∇_naive(O)(i) = O(i+1) — shift index
   - (p₀^{9.12}, q₀^{9.12}) = (p', q') - Sign(∇_naive(O))
   - O'(1) = O(2) + (p₀^{9.12}, q₀^{9.12}), O'(i) = O(i+1) for i ≥ 2
3. ˡSign(O') = Σ_{even i} (p'_i, q'_i) + Σ_{odd i} (q'_i, p'_i)

### Recursive AC computation (equation 11.2)

**Base case** (|Ǒ| = 0):
```
L_τ = (1, 0)_★   if γ = B⁺
    = (0, -1)_★   if γ = B⁻
    = (0, 0)_★    otherwise
```

**Induction** (|Ǒ| > 0):
```
L_τ = ϑ̂^{s_τ, O}(L_{τ'}) ⊗ (0, ε_τ)         if ★ ∈ {B, D}
    = ϑ̂^{s_τ, O}(L_{τ'} ⊗ (ε_℘, ε_℘))       if ★ ∈ {C, C̃}
```
where τ' = ∇(τ), and:
- For B/D: sign twist ⊗(0, ε_τ) applied AFTER lift, only at top level
- For C/C̃: det twist applied BEFORE lift (for ℘=∅: ε_℘=0, no twist)

## Verification

Sign(L_τ) = (p_τ, q_τ) verified for all D, M, C types (152+ DRCs).
B type has remaining ±1 offset issue related to B⁺/B⁻ convention in multi-step chains.
