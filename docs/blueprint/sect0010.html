3.2 Shape-shift for C type (case analysis)

The C-type shape shift is more delicate: it modifies only column 0 of \(\mathcal{P}\) (the roles of \(\mathcal{P}\) and \(\mathcal{Q}\) do not swap, unlike M type). The construction depends on the value of \(\mathcal{P}_\tau (\mathbf{c}_1(\iota _\wp ), 1)\), giving four sub-cases.

3.2.1 Setup

Let \(\tau \in \mathrm{PBP}_C(\check{\mathcal{O}}, \wp )\) with \((1, 2) \in \mathrm{PP}_\star (\check{\mathcal{O}})\) and \((1, 2) \notin \wp \). Set

\[ k := \mathbf{c}_1(\iota _\wp ),\qquad k_\uparrow := \mathbf{c}_1(\iota _{\wp _\uparrow }), \]

where \(\wp _\uparrow := \wp \cup \{ (1, 2)\} \). By the PP assumption, \(k_\uparrow {\gt} k \geq 1\).

Since \(\)\(\) is unchanged, \(\)τ_.Q(i, j) := τ.Q(i, j)\(\) for all \((i, j)\).

3.2.2 Case analysis for the new paint

The construction of \(\mathcal{P}_{\tau _\uparrow }\) splits into four sub-cases based on the value of \(\mathcal{P}_\tau (k, 1)\). Each sub-case is a separate definition corresponding to its own Lean helper.

3.2.3 Validity and bijectivity

Lemma 27 Shape shift C produces a valid PBP
✓
#

Uses

  • Definition 2
  • Definition 22

Lean declarations

  • shapeShiftC

The painted diagram produced by any of the four sub-cases of Def. 22 satisfies all PBP axioms: the allowed symbol sets (sym_P, sym_Q), layer monotonicity, the dot_match invariant, and the row / column uniqueness conditions (row_s, row_r, col_c_P, col_c_Q, col_d_P, col_d_Q).

Proof ▶

We check each axiom case-by-case.

sym_P: The new paint function outputs values only in \(\{ r, c, d\} \) (the forced values in the modified region of column 1) or \(\mathcal{P}_\tau (i, j)\) (unchanged cells, inheriting validity from \(\tau \)). Both lie in the C-type \(\mathcal{P}\)-allowed set \(\{ \bullet , s, r, c, d\} \). Formalized as shiftPaintP_C_concrete_case_a1_allowed and the analogous a2, b2 variants.

sym_Q: \(\mathcal{Q}\) is unchanged, so the axiom carries over directly from \(\tau \).

layer monotonicity: In case (a.1) the sequence \(\cdots r\ c\ d\) has monotone \(\mathrm{layerOrd}\) (\(2 \le 3 \le 4\)); the pattern is preserved across sub-cases by construction of \(\mathcal{P}_{\tau _\uparrow }\).

col_c_P, col_d_P: Case (a.1) moves the pre-existing \(c\) at row \(k-1\) to the new position \(k_\uparrow - 1\) (rewriting the old cell as \(r\)), hence column 1 still has exactly one \(c\). The newly-introduced \(d\) sits at row \(k_\uparrow \); uniqueness holds because the original tail contained no \(d\) at the top (by layer monotonicity on \(\tau \)). Formalized as shiftPaintP_C_concrete_case_a2_col_c and shiftPaintP_C_concrete_case_a2_col_d.

row_s, row_r: Same-row uniqueness carries over from \(\tau \) since the C-type \(\mathcal{Q}\) only contains \(\{ \bullet , s\} \), so the modifications to \(\mathcal{P}\)’s column 1 cannot conflict row-wise with anything in \(\mathcal{Q}\).

dot_match: The dot positions of \(\mathcal{P}_{\tau _\uparrow }\) agree with those of \(\mathcal{Q}_{\tau _\uparrow } = \mathcal{Q}_\tau \) because the modifications only fill non-dot cells in the strictly extended tail region \([k, k_\uparrow ]\).

The Lean shapeShiftC definition exhibits all of the above by constructing the final PBP with every axiom verified.
