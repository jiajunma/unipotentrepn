3.2 Shape-shift for C type (case analysis)

The C-type shape shift is more delicate: it modifies only column 0 of \(\mathcal{P}\) (the roles of \(\mathcal{P}\) and \(\mathcal{Q}\) do not swap, unlike M type). The construction depends on the value of \(\mathcal{P}_\tau (\mathbf{c}_1(\iota _\wp ), 1)\), giving four sub-cases.

3.2.1 Setup

Let \(\tau \in \mathrm{PBP}_C(\check{\mathcal{O}}, \wp )\) with \((1, 2) \in \mathrm{PP}_\star (\check{\mathcal{O}})\) and \((1, 2) \notin \wp \). Set

\[ k := \mathbf{c}_1(\iota _\wp ),\qquad k_\uparrow := \mathbf{c}_1(\iota _{\wp _\uparrow }), \]

where \(\wp _\uparrow := \wp \cup \{ (1, 2)\} \). By the PP assumption, \(k_\uparrow {\gt} k \geq 1\).

Since \(\)\(\) is unchanged, \(\)τ_.Q(i, j) := τ.Q(i, j)\(\) for all \((i, j)\).

3.2.3 Validity and bijectivity

Remark 23
#
τ_\(\) is a valid PBP] The painted diagram \(\tau _\uparrow \) produced by any of the four sub-cases satisfies all PBP axioms (sym_P, mono_P, col_c_P, col_d_P, row_s, row_r). This is implicit in the Lean shapeShiftC construction: the new paint values are always in the C-type \(\mathcal{P}\)-allowed set \(\{ \bullet , s, r, c, d\} \); the \(r \to c \to d\) layer ordering is preserved (case (a.1) moves the \(c\) to a new position, with the old \(c\) becoming \(r\)); row uniqueness carries over since the C-type \(\mathcal{Q}\) only contains \(\{ \bullet , s\} \). Details: [BMSZb] p. 66.
